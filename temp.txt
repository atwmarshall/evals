Step 1 — Generate variations, validate, run, report

  uv run python scripts/sensitivity.py \
    --dataset datasets/challenge2/extraction.jsonl \
    --scorer schema \
    --model llama3.2:3b \
    --variations synonym_swap rephrase \
    --limit 5 \
    --validation-threshold 0.7

  This exercises the full pipeline: generates 2 variation types on 5 samples, validates with llama3.2:3b as judge,
  runs schema scorer across all 3 datasets (baseline + 2), prints tables, saves to results/sensitivity/ and
  datasets/generated/sensitivity/.

  Step 2 — Inspect the results

  # Find the saved dir and inspect it
  uv run python scripts/show.py results/sensitivity/$(ls -t results/sensitivity/ | head -1)/$(ls -t
  results/sensitivity/$(ls -t results/sensitivity/ | head -1)/ | head -1)

  Or grab the path printed at the end of Step 1 and pass it directly:
  uv run python scripts/show.py results/sensitivity/2026-03-15/143022_extraction_schema

  # show only unstable samples
  uv run python scripts/show.py results/sensitivity/2026-03-15/143022_extraction_schema -f

  # drill into one sample
  uv run python scripts/show.py results/sensitivity/2026-03-15/143022_extraction_schema --id c2-001

  Step 3 — Reuse saved variations (skip the LLM generation step)

  # Find the generated dir printed at the end of Step 1, then:
  uv run python scripts/sensitivity.py \
    --dataset datasets/challenge2/extraction.jsonl \
    --scorer exact \
    --model llama3.2:3b \
    --reuse-variations datasets/generated/sensitivity/2026-03-15_extraction_ministral-3_3b \
    --no-save-variations

  This skips generation entirely, loads the saved variations, runs the exact scorer instead of schema — so you can
  compare scorer sensitivity across scorer types on the same input variations.

  Step 4 — Trigger the model separation guard

  VARIATION_MODEL=llama3.2:3b uv run python scripts/sensitivity.py \
    --dataset datasets/challenge2/extraction.jsonl \
    --scorer schema \
    --variations synonym_swap

  Should immediately exit with an error about VARIATION_MODEL equalling JUDGE_MODEL.

  ---
  What to watch for:
  - rephrase will likely show the highest discard rate in validation (the model sometimes returns JSON instead of a
  rewritten question, as you saw when you ran the temp script)
  - synonym_swap tends to survive validation better
  - Schema scorer will have low sensitivity (it's structural) — if you repeat with --scorer judge, you'll see much
  higher variance
